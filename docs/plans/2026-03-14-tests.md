# Tests

**Date:** 2026-03-14  
**Status:** Done  
**Priority:** Medium

## Problem

A small unit-test baseline exists (config, tool presentation, auth store) but
coverage of the agent loop and provider wire format is still missing. The areas
most likely to break silently are:

- Provider wire-format serialisation (tool-call payloads, role names, etc.)
- Tool implementations (`bash` exit-code handling, `edit` exact-match logic,
  `read_file` offset/limit arithmetic)
- The agent loop (`run_agent_loop` turn counting, `before_tool_call` blocking,
  truncation on `max_turns`)
- `ui.rs` line-wrapping and scroll arithmetic

## Strategy

Use Rust's built-in `#[test]` / `#[tokio::test]` framework — no external test
harness needed. Tests live in the same crate, either inline or in
`src/*/tests.rs` submodules. No mocking framework; use simple hand-rolled
fakes.

## Priority areas

### 2. Agent loop (integration tests with a fake provider)

Define a `MockProvider` that returns a canned sequence of `LlmEvent`s:

```rust
struct MockProvider {
    turns: Vec<Vec<LlmEvent>>,
}

impl LlmProvider for MockProvider {
    fn stream_chat_with_tools(&self, …) -> LlmStream { … }
}
```

```rust
#[tokio::test]
async fn agent_loop_single_text_turn() {
    // Provider returns Token("hello") + Done.
    // Expect: Done event, messages has one assistant message.
}

#[tokio::test]
async fn agent_loop_tool_call_then_text() {
    // Turn 1: ToolCall { bash, "echo hi" } + Done.
    // Turn 2: Token("the result is: hi") + Done.
    // Expect: ToolCallStart + ToolCallEnd + TextToken + Done events.
}

#[tokio::test]
async fn agent_loop_max_turns_reached() {
    // Provider always returns a tool call; loop should stop at max_turns
    // and emit AgentEvent::Error.
}

#[tokio::test]
async fn agent_loop_before_hook_can_block() {
    // before_tool_call returns false; expect ToolResult with is_error.
}
```

### 3. Message serialisation (unit tests)

Verify `Role::as_str()` values and the OpenAI tool-call serialisation in
`openai.rs` round-trip correctly. These are the most likely source of
silent API failures when the provider changes.

```rust

#[test]
```
